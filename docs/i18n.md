# i18n 전략

건담 카드게임 덱빌더 프론트엔드 다국어 지원 구현 계획 및 현황.

---

## 지원 언어

| URL 세그먼트 | i18next 로케일 | 표시 이름 |
|---|---|---|
| `/ko/...` | `ko` | 한국어 |
| `/en/...` | `en` | English |
| `/jp/...` | `ja` | 日本語 |

- 소스 언어: **영어** (fallback `en`)
- URL에서는 일본어를 `jp`로 표기하지만 i18next 내부에서는 `ja`로 매핑 (`LOCALE_MAP`)

---

## 라이브러리

- `i18next` — 번역 엔진
- `react-i18next` — React 훅 (`useTranslation`)
- `i18next-browser-languagedetector` — localStorage 언어 감지

---

## URL 구조

TanStack Router의 `$locale` 동적 파라미터를 사용한 URL 기반 라우팅.

```
/ko/cardlist          한국어 카드 목록
/en/cardlist          영어 카드 목록
/jp/cardlist          일본어 카드 목록
/ko/deck/:deckId      한국어 덱 상세
```

`src/routes/$locale.tsx` 레이아웃 라우트가 URL의 locale을 읽어 `i18n.changeLanguage()`를 호출한다.
루트(`/`) 접근 시 `DEFAULT_LOCALE("ko")`로 리다이렉트.

## 구현 패턴

### React 컴포넌트 내부
```tsx
const { t } = useTranslation("common");
t("nav.cardSearch")
```

### 컴포넌트 외부 (render/ 함수 등)
```ts
import i18n from "@/i18n";
i18n.t("keyword.ACTION", { ns: "game" })
```

### 동적 키 (TypeScript 엄격 모드 우회)
```ts
(i18n.t as any)(key, { ns: "filters" }) as string
```

### 언어 전환
```ts
import { LOCALE_MAP } from "@/i18n";
i18n.changeLanguage(LOCALE_MAP[locale]); // locale: "ko" | "en" | "jp"
```

## 남은 작업

### 언어 전환 UI
`src/components/LanguageSwitcher.tsx` 생성 후 헤더 또는 사이드바에 추가.
`Link to="/$locale/..." params={{ locale: nextLocale }}` 방식으로 URL을 바꾸면 `$locale.tsx`가 자동으로 언어를 전환한다.

### 미번역 인라인 문자열
컴포넌트 내 한국어 하드코딩 문자열들은 아직 `t()` 호출로 교체되지 않았다.
`common.json`에 키를 추가하고 `useTranslation`으로 교체한다.

### RulesPage
가장 많은 텍스트량. `rules` 네임스페이스 JSON 파일을 신규 생성해서 추출 예정.
