# i18n 패턴 및 적용 범위

## 네임스페이스

| 네임스페이스 | 용도                                                         |
| ------------ | ------------------------------------------------------------ |
| `common`     | UI 공통 레이블 (버튼, 탭, 헤더 등)                           |
| `game`       | 게임 용어 (카드 타입, 키워드, 존, 컬러, 트레이트, 시리즈 등) |
| `filters`    | 필터 UI 전용 레이블 (정렬, 팩 그룹 등)                       |
| `rules`      | 룰 페이지 텍스트                                             |

locale 파일 위치: `src/locales/{ko,en,ja}/{namespace}.json`

새 문자열 추가 시 **ko, en, ja 세 파일 모두** 동시에 추가한다.

## 컴포넌트에서 문자열 표시

```tsx
// Hook — 언어 변경 시 자동 re-render
const { t } = useTranslation("game");
t("kind.UNIT"); // → "유닛" / "Unit"
t("area.battle"); // → "배틀 에어리어" / "Battle Area"
```

## 카드 이름 (LocalizedString)

```tsx
// LocalizedString = { en: string; ko: string; [lang: string]: string }
import { useLocalize } from "@/lib/localize"; // 컴포넌트용 hook
import { localize } from "@/lib/localize"; // 비-hook 컨텍스트용

const localize = useLocalize();
localize(card.name); // 현재 언어로 자동 선택
localize(card.pilot?.name); // null-safe
```

## 정적 레이블 (필터 등) — 반드시 render body 안에서 호출

```tsx
// ❌ 모듈 최상위에서 호출 — 언어 변경에 반응 안 함
const labels = getKeywordLabels();

// ✓ 컴포넌트 body 안에서 호출
function FilterControls() {
  const { t } = useTranslation("common"); // re-render 트리거
  const keywordLabels = getKeywordLabels(); // 매 render마다 현재 언어로 계산
  const seriesLabels = getSeriesLabels();
  const packGroups = getPackGroups();
  // getKindLabel(k), getZoneLabel(z), getColorLabel(c) — 단건 조회도 동일
}
```

`filterConstants.ts`의 helper 함수들은 내부에서 `i18n.t()`를 호출하므로 render body 안에서 쓰면 자동으로 현재 언어를 사용한다.

## locale 라우팅 (`$locale.tsx`)

- `beforeLoad`에서 `i18n.changeLanguage()` 호출
- **`cause !== "preload"` 조건 필수** — 없으면 TanStack Router가 링크 hover 시 preload 목적으로 `beforeLoad`를 실행해, 언어가 번갈아 바뀌며 화면이 깜빡인다

```ts
beforeLoad: ({ params, cause }) => {
  if (cause !== "preload") {
    const i18nLocale = LOCALE_MAP[params.locale];
    if (i18nLocale && i18n.language !== i18nLocale) {
      i18n.changeLanguage(i18nLocale);
    }
  }
},
```

## 키워드 팝업 설명 (`keywordDescription.ts`)

```ts
function prose(ko: string, en: string): DescriptionLine {
  return [{ type: "prose", en, ko }];
}
```

- **인자 순서**: `prose(ko, en)` — ko가 첫 번째
- en과 ko를 반드시 구분해서 작성. 같은 한국어 텍스트를 `en`에 넣으면 영어 모드에서 한국어가 표시됨

## 카드 description 구조

`3.processed.json`의 `description` 필드는 `Array<Array<token>>` (줄별 배열의 배열).  
`{ tokens: [...] }` 형태가 **아니다**.

```ts
// ✓ 올바른 iteration
for (const line of card.description as Array<Array<{ type: string; ko?: string }>>) {
  if (!Array.isArray(line)) continue;
  line.forEach(token => ...);
}

// ❌ 잘못된 패턴 — line.tokens는 항상 undefined
for (const line of card.description as Array<{ tokens: [...] }>) {
  line.tokens?.forEach(...); // 동작 안 함
}
```

## 키워드 누락 감사

locale `game.json`의 `keyword.*` 키와 `filterConstants.ts`의 `ALL_KEYWORDS`는 반드시 **실제 카드 데이터에 존재하는 키워드**와 일치해야 한다.

새 키워드를 추가하거나 기존 키워드가 의심스러울 때:

```bash
# 카드 데이터에서 해당 키워드가 실제로 사용되는지 확인
grep -c "KEYWORD_NAME" /Users/ben/git/personal/GCG_DeckBuilder/data/3.processed.json
```

결과가 0이면 아래 파일에서 모두 제거:

1. `src/locales/ko|en|ja/game.json` → `keyword.KEYWORD_NAME`
2. `src/lib/filterConstants.ts` → `ALL_KEYWORDS` 배열
3. `src/render/keywordDescription.ts` → 팝업 설명 항목
4. `src/components/CardDescription.tsx` → `TRIGGER_STYLES` + `TRIGGER_KEYWORD_MAP`
5. `src/routes/$locale/cardlist.tsx`, `deck/$deckId.tsx` → URL 파라미터 유효성 목록

**사례**: `END_OF_TURN`("내 턴이 끝날 때") — 카드 데이터 0건 확인 후 전체 제거 (2026-04-12)
